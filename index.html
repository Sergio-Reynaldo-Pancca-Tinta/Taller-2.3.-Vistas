📌 Introducción: Se implementaron vistas sobre una base de datos clínica (tPaciente, tMedico, tEspecialidad, tEnfermedad, tDiagnostico) en Oracle SQL Developer. Las vistas permiten abstraer consultas repetitivas, optimizar el acceso a información médica y mejorar la seguridad. El informe cubre 7 requerimientos (desde médicos con especialidades hasta pacientes sin enfermedades asociadas), siguiendo normativa APA 7.

1 5.1 Médicos y especialidades
🔍 Crear vista que muestre TODOS los datos del médico y sus especialidades.
create or replace view VistasMedicoEspecialidad as
select m.CodM, m.TipoDocM, m.NroDocM,
       m.PaternoM, m.MaternoM, m.NombresM,
       m.CelularM, m.FechaNacimientoM, m.GeneroM,
       e.CodEsp, e.NombreEsp, e.DescripcionEsp,
       em.FechaDeObtencionDeLaEspecialidadEspM
from tMedico m
join tEspecialidadDelMedico em on m.CodM = em.CodM
join tEspecialidad e on em.CodEsp = e.CodEsp;
✅ Vista creada. Para mostrar los datos: SELECT * FROM VistasMedicoEspecialidad; — Muestra todos los médicos con el detalle de cada especialidad.
2 5.2 Conteo por médico
📊 A partir de la vista anterior: mostrar cuántas especialidades tiene cada médico.
select CodM, PaternoM, MaternoM, NombresM,
       count(CodEsp) as NumeroDeEspecialidades
from VistasMedicoEspecialidad
group by CodM, PaternoM, MaternoM, NombresM
order by NumeroDeEspecialidades desc;
📌 Resultado esperado: M1 → 1 especialidad, M2 → 2 especialidades.
3 5.3 Médicos ≥ 2 especialidades
🏥 Mostrar médicos que tengan 2 o más especialidades.
select CodM, PaternoM, MaternoM, NombresM,
       count(CodEsp) as NumeroDeEspecialidades
from VistasMedicoEspecialidad
group by CodM, PaternoM, MaternoM, NombresM
having count(CodEsp) >= 2;
⚕️ El Dr. M2 (Alejandra Cabral) posee Pediatría y Cardiología → cumple condición.
4 5.4 Enfermedades sin diagnóstico
🦠 Vista que muestre enfermedades aún no detectadas en diagnósticos.
create or replace view VistasEnfermedadesNoDetectadas as
select e.CodE, e.NombreE, e.DescricionE
from tEnfermedad e
left join tEnfermedadEnElDiagnostico ed
    on e.CodE = ed.CodE
where ed.CodE is null;
⚠️ Según datos: "Artritis (E3)" nunca ha sido diagnosticada.
5 5.5 Especialidades sin médico
🏛️ Vista de especialidades para las cuales aún no hay médicos.
create or replace view VistasEspecialidadesSinMedico as
select e.CodEsp, e.NombreEsp, e.DescripcionEsp
from tEspecialidad e
left join tEspecialidadDelMedico em
    on e.CodEsp = em.CodEsp
where em.CodEsp is null;
🦷 Esp4 (Odontología) no tiene ningún médico asignado en la clínica.
6 5.6 Conteo de especialidades sin médico
🔢 A partir de la vista anterior: mostrar cuántas especialidades no tienen médico.
select count(*) as TotalEspecialidadesSinMedico
from VistasEspecialidadesSinMedico;
📆 Resultado: 1 especialidad (Odontología) aún no tiene médico.
7 5.7 Pacientes sin enfermedad asociada
👤 Vista con nombres de pacientes cuyos diagnósticos NO estén relacionados con enfermedad alguna.
create or replace view VistasPacientesSinEnfermedad as
select p.CodP, p.PaternoP, p.MaternoP, p.NombresP,
       d.CodD, d.DescripcionD, d.FechaHoraD
from tPaciente p
join tDiagnostico d on p.CodP = d.CodP
left join tEnfermedadEnElDiagnostico ed on d.CodD = ed.CodD
where ed.CodD is null;

-- Nombres de pacientes:
select distinct p.PaternoP||' '||p.MaternoP||', '||p.NombresP as NombreCompleto
from tPaciente p
join tDiagnostico d on p.CodP = d.CodP
left join tEnfermedadEnElDiagnostico ed on d.CodD = ed.CodD
where ed.CodD is null;
👶 La paciente Elena Pérez (P02) tiene diagnóstico "Niña sana" sin enfermedades vinculadas.

📌 Conclusiones

La implementación de vistas en Oracle SQL Developer permitió simplificar consultas complejas y agilizar el acceso a información clave de la clínica, como la relación médicos-especialidades, enfermedades no diagnosticadas y pacientes sin asociación patológica. Se comprobó que las vistas no solo facilitan la reutilización de lógica de negocio, sino que también mejoran la seguridad al ocultar detalles estructurales de la base de datos.

💡 Sugerencias

✔ Incorporar índices sobre las columnas utilizadas en los JOIN (codM, codEsp, codE, codD) para optimizar el rendimiento cuando la base de datos crezca en volumen.
✔ Considerar vistas materializadas si se requieren consultas frecuentes con agregaciones.
✔ Establecer permisos diferenciados por roles (médico, administrador, paciente) para garantizar confidencialidad de datos clínicos.
